### 💡 O que é esse projeto?
Criei um banco de dados fictício de e-commerce para praticar SQL de forma mais próxima do mundo real.
As consultas cobrem desde análises de vendas até automações com triggers e procedures.

### 📌 O que tem em cada pasta?

### 🔁 Ctes-Subqueries

Consultas usando WITH (CTEs) e subqueries. Aprendi que isso deixa o código muito mais organizado e fácil de entender!

### 🪟 Window-functions/

Funções como RANK(), LAG() e SUM() OVER — perfeitas pra análises de tendência e ranking de vendas.

### ⚙️ Stored-Procedures
Procedures em PL/pgSQL pra automatizar processos, como atualização de estoque.

### 🔔 Triggers
Gatilhos que disparam automaticamente quando algo muda no banco — usei bastante pra auditoria e validação.

### 📊 Analise de Vendas
As queries que mais gosto! Aqui calculo KPIs como receita, ticket médio e produtos mais vendidos.

### ⚡Performance-indices
Ainda estou aprendendo bastante sobre otimização, mas já tenho exemplos com EXPLAIN ANALYZE e índices compostos.

### 🛠️ Tecnologias usadas

- PostgreSQL — banco de dados principal
- PL/pgSQL — linguagem para procedures e triggers
- DBeaver — ferramenta que uso pra gerenciar o banco
- Git & GitHub — pra versionar e compartilhar o projeto
